## 问题一：顾客登录 401 循环跳转

### 问题描述

顾客登录后页面不断跳转到登录页，无法正常进入系统。

### 原因分析

1. 前端 API 拦截器对所有 401 响应都执行跳转登录逻辑
2. 顾客登录接口 `/api/customer/login` 和管理员登录接口 `/api/admin/login` 都会返回 401（登录失败时）
3. 登录成功后，后端返回的用户信息格式与前端期望不一致

### 解决方案

**修改文件**: `frontend/src/api/index.js`

```javascript
api.interceptors.response.use(
  response => response,
  error => {
    if (error.response && error.response.status === 401) {
      const url = error.config.url || ''
      // 只对非登录接口进行 401 跳转
      if (!url.includes('/login')) {
        sessionStorage.removeItem('user')
        window.location.href = '/login'
      }
    }
    return Promise.reject(error)
  }
)
```

## 问题二：数据库字段不匹配

### 问题描述

预约时报错：`Unknown column 'room_id' in 'booking'`

### 原因分析

数据库 booking 表最初没有 room_id 字段，但前端和后端代码尝试操作该字段。

### 解决方案

**1. 数据库添加字段**:
```sql
ALTER TABLE hotel_management.booking ADD COLUMN room_id BIGINT AFTER guest_id;
```

**2. 后端 Booking.java 添加字段**:
```java
private Long roomId;
```

**3. BookingMapper.xml 更新 INSERT 语句**:
```xml
<insert id="insert">
    INSERT INTO booking (guest_id, room_id, check_in_date, check_out_date, status, special_requests)
    VALUES (#{guestId}, #{roomId}, #{checkInDate}, #{checkOutDate}, #{status}, #{specialRequests})
</insert>
```

## 问题四：管理员确认预订报错"未知错误"

### 问题描述

点击预订管理的"确认"按钮后，提示"确认失败: 未知错误"。

### 原因分析

1. 后端 `confirmBooking` 方法返回 `int`，但没有异常处理
2. 当前端收到异常响应时，显示的是后端返回的默认错误信息
3. MyBatis 执行时如果出错，异常信息被吞掉

### 解决方案

**1. 修改 BookingController.java - 添加详细错误处理**:
```java
@PatchMapping("/{id}/confirm")
public ResponseEntity<?> confirmBooking(@PathVariable Long id) {
    try {
        int result = bookingService.confirmBooking(id);
        if (result > 0) {
            return ResponseEntity.ok(Map.of("message", "预订已确认", "id", id));
        } else {
            return ResponseEntity.badRequest().body(Map.of("message", "预订不存在或确认失败"));
        }
    } catch (Exception e) {
        e.printStackTrace();  // 输出详细日志
        return ResponseEntity.badRequest().body(Map.of("message", "确认失败: " + e.getMessage()));
    }
}
```

**2. 修改 BookingService.java - 添加 @Transactional 和异常抛出**:
```java
@Transactional
public int confirmBooking(Long id) {
    Booking booking = bookingMapper.findById(id);
    if (booking == null) {
        throw new RuntimeException("预订不存在");
    }
    // ... 业务逻辑
}
```

## 问题五：顾客端预约时"房间ID不能为空"

### 问题描述

顾客选择房间后点击预订，提示"房间ID不能为空"。

### 原因分析

`CustomerRoom.vue` 的 `handleBooking` 方法正确设置了 `roomId`，但 `handleSubmit` 中的 `api.addBooking` 调用没有包含 `roomId` 参数。

### 解决方案

**修改 CustomerRoom.vue**:
```javascript
handleSubmit() {
  // 添加 roomId 校验
  if (!this.bookingForm.roomId) {
    ElMessage.warning('请选择房间')
    return
  }
  // ...
  api.addBooking({
    guestId: guestId,
    roomId: this.bookingForm.roomId,  // 添加 roomId
    checkInDate: this.bookingForm.checkInDate,
    checkOutDate: this.bookingForm.checkOutDate,
    status: 'pending'
  })
}
```

## 问题六：确认预订后自动生成清洁任务

### 需求描述

管理员确认预订后，自动为该房间创建清洁任务。

### 实现方案

在 `BookingService.confirmBooking` 方法中添加任务创建逻辑：

```java
@Transactional
public int confirmBooking(Long id) {
    Booking booking = bookingMapper.findById(id);
    if (booking == null) {
        throw new RuntimeException("预订不存在");
    }
    int result = bookingMapper.updateStatus(id, "confirmed");

    // 自动创建清洁任务
    if (result > 0 && booking.getRoomId() != null) {
        Room room = roomMapper.findById(booking.getRoomId());
        String priority = "normal";
        // 套房优先级设为高
        if (room != null && "suite".equals(room.getRoomType())) {
            priority = "high";
        }
        HousekeepingTask task = new HousekeepingTask();
        task.setRoomId(booking.getRoomId());
        task.setTaskType("cleaning");
        task.setPriority(priority);
        task.setStatus("pending");
        task.setNotes("预约入住前清洁");
        taskMapper.insert(task);
    }
    return result;
}
```

## 经验总结

1. **前后端字段一致性**: 每次修改数据库结构后，需要同步更新后端 Entity、Mapper XML、前端 API 调用
2. **异常处理**: 后端接口应使用 try-catch 包装业务逻辑，并返回明确的错误信息
3. **日志输出**: 关键位置添加 `e.printStackTrace()` 便于排查问题
4. **事务管理**: 多数据操作使用 `@Transactional` 确保数据一致性
