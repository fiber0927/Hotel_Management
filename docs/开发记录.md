## 三、房间状态流转 (v1.2.0+)

### 3.1 状态定义

| 状态值 | 显示文本 | 含义 | 预订按钮 |
|--------|----------|------|----------|
| available | 可预订 | 空闲房间 | 可用 |
| booked | 已被预约 | 已被预约，尚未入住 | **禁用** |
| occupied | 已占用 | 正在使用中 | **禁用** |
| cleaning | 清洁中 | 退房后待清洁 | **禁用** |

### 3.2 状态流转规则

```
                    顾客提交预订
                   ─────────────────→
                  │                    │
[可预订] ←─────────────────────────── [已被预约]
   ↑│                              ││
   ││ 管理员拒绝                   ││ 管理员确认
   ││                              ││
   ││                              ▼
   ││                         [已入住]
   ││                              │
   ││ 退房                         │
   ││                              │
   ││                              ▼
   ││                        [已占用]
   ││                              │
   ││ 清洁完成                     │
   ││                              │
   ││ 管理员确认退房               │
   ││                              │
   └────────────────────────────────┘
```

### 3.3 核心业务规则（重要）

**房间释放规则**：
- 顾客删除已确认的预订 → **不释放房间**，房间保持"已被预约"状态
- 只有通过完整流程才能释放房间：**入住 → 退房 → 管理员确认退房**
- 防止恶意删单重新抢房

### 4.2 注册校验

| 字段 | 校验规则 |
|------|----------|
| 用户名 | 3-20字符，不能为空，不能重复 |

| 确认密码 | 必须与密码一致 |

### 4.3 后端接口

**POST /api/customer/register**


## 五、预订手机号自动填充 (v1.3.0)

### 5.1 功能说明

顾客登录后，在预订房间时，联系电话输入框自动填充为注册时填写的手机号，支持手动修改。

### 5.2 实现方式

登录时将用户手机号存入 sessionStorage：
```javascript
const userData = {
    id: res.data.id,
    username: res.data.username,
    name: res.data.name || '',
    phone: res.data.phone || '',  // 新增
    role: 'customer'
}
sessionStorage.setItem('user', JSON.stringify(userData))
```

预订时读取并填充：
```javascript
handleBooking(room) {
    const user = JSON.parse(sessionStorage.getItem('user') || '{}')
    this.bookingForm = {
        roomId: room.id,
        roomNumber: room.roomNumber,
        checkInDate: '',
        checkOutDate: '',
        phone: user.phone || ''  // 自动填充
    }
    this.bookingDialog = true
}
```

## 六、历史 Bug 修复

| Bug | 原因 | 修复方案 |
|-----|------|----------|
| 401循环跳转 | API拦截器对登录接口也跳转 | 登录接口排除判断 |
| booking表缺room_id | 数据库字段缺失 | ALTER TABLE添加 |
| 确认预订报未知错误 | 无异常处理 | 添加try-catch |
| 房间状态不同步 | 各操作未联动更新状态 | 事务中自动更新状态 |
| 删除预订释放房间 | 错误释放已确认的房间 | 删除时不改变房间状态 |
| 预订手机号未自动填充 | 未保存用户手机号 | 登录时存储，注册时读取 |
