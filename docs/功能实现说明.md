## 一、顾客端登录与预约流程

### 1.1 账号角色区分

系统通过两个独立的登录接口区分用户角色：

| 角色 | 登录接口 | 路由前缀 |
|------|----------|----------|
| 顾客 | POST /api/customer/login | /customer/* |
| 管理员 | POST /api/admin/login | /main/* |

前端 API 层封装：

```javascript
// frontend/src/api/index.js
customerLogin(username, password) {
  return api.post('/customer/login', { username, password })
},
login(username, password) {
  return api.post('/admin/login', { username, password })
}
```

### 1.2 预约提交逻辑

顾客预约流程：

```
1. 顾客浏览房间列表 (GET /api/rooms)
2. 点击"立即预订"按钮，选择房间和日期
3. 提交预约请求 (POST /api/bookings)
   - 同时创建 Guest 信息 (客人姓名、电话)
4. 预约状态默认为 "pending" (待处理)
5. 等待管理员确认
```

关键代码 - CustomerRoom.vue:

```javascript
handleSubmit() {
  // 1. 验证表单
  // 2. 创建客人信息
  api.addGuest({
    name: user.name || user.username || '顾客',
    phone: this.bookingForm.phone
  }).then(res => {
    const guestId = res.data.id
    // 3. 创建预约
    api.addBooking({
      guestId: guestId,
      roomId: this.bookingForm.roomId,
      checkInDate: this.bookingForm.checkInDate,
      checkOutDate: this.bookingForm.checkOutDate,
      status: 'pending'
    })
  })
}
```

## 二、管理员端预订管理与客房服务

### 2.1 确认预订接口

管理员确认预订时，系统执行以下操作：

```
1. 更新预订状态：pending -> confirmed
2. 自动创建客房清洁任务
3. 返回操作结果
```

后端实现 - BookingService.java:

```java
@Transactional
public int confirmBooking(Long id) {
    Booking booking = bookingMapper.findById(id);
    if (booking == null) {
        throw new RuntimeException("预订不存在");
    }
    // 更新状态为已确认
    int result = bookingMapper.updateStatus(id, "confirmed");

    // 自动创建清洁任务
    if (result > 0 && booking.getRoomId() != null) {
        Room room = roomMapper.findById(booking.getRoomId());
        String priority = "normal";
        // 套房优先级设为高
        if (room != null && "suite".equals(room.getRoomType())) {
            priority = "high";
        }
        HousekeepingTask task = new HousekeepingTask();
        task.setRoomId(booking.getRoomId());
        task.setTaskType("cleaning");
        task.setPriority(priority);
        task.setStatus("pending");
        task.setNotes("预约入住前清洁");
        taskMapper.insert(task);
    }
    return result;
}
```

### 2.2 客房服务任务类型

| 任务类型 | taskType | 说明 |
|----------|----------|------|
| 清洁 | cleaning | 房间清洁打扫 |
| 维修 | repair | 设施设备维修 |

| 优先级 | priority | 触发条件 |
|--------|----------|----------|
| 高 | high | 套房(suite)类型的房间 |
| 普通 | normal | 标准间(standard) |
| 低 | low | 可手动设置 |

## 三、房间状态与预约同步机制

### 3.1 房间状态说明

| 状态 | status | 含义 |
|------|--------|------|
| available | 可用 | 可预订、可入住 |
| booked | 已预订 | 已被预约，尚未入住 |
| occupied | 已占用 | 正在使用中 |
| cleaning | 清洁中 | 退房后待清洁 |
| maintenance | 维修中 | 设施维修 |

### 3.2 预约流程中的状态变化

```
顾客浏览房间
    ↓ (查看可预订房间 GET /api/rooms/available)
房间列表显示 available 状态
    ↓ (提交预约)
预约状态: pending
    ↓ (管理员确认)
预约状态: confirmed
房间状态: booked (表示已被预约)
    ↓ (顾客办理入住)
入住状态: checked_in
房间状态: occupied
    ↓ (顾客退房)
房间状态: cleaning (自动生成清洁任务)
    ↓ (清洁完成)
房间状态: available
```

### 3.3 自动任务生成

当预订被确认时，自动创建清洁任务：

```sql
-- 确认预订后自动执行
INSERT INTO housekeeping_task (room_id, task_type, priority, status, notes)
VALUES (#{roomId}, 'cleaning', 'high/normal', 'pending', '预约入住前清洁')
```

## 四、核心 API 接口一览

### 4.1 预订相关

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/bookings | 获取所有预订 |
| GET | /api/bookings/{id} | 获取单个预订 |
| POST | /api/bookings | 创建预订 |
| PUT | /api/bookings/{id} | 更新预订 |
| PATCH | /api/bookings/{id}/confirm | 确认预订 |
| PATCH | /api/bookings/{id}/cancel | 取消预订 |

### 4.2 客房服务相关

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/housekeeping | 获取所有任务 |
| POST | /api/housekeeping | 创建任务 |
| PATCH | /api/housekeeping/{id}/complete | 完成任务 |
| DELETE | /api/housekeeping/{id} | 删除任务 |
